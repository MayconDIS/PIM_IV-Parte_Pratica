Resumo e Abstract

Resumo

O Nex_TI é um sistema web educacional focado nas metodologias de Estudo Ativo, Repetição Espaçada (Algoritmo SM-2) e Gamificação. O objetivo é tornar a aprendizagem tecnológica mais envolvente, recompensando o aluno com Pontos de Experiência (XP) e Moedas (Coins) à medida que progride. O projeto integra conceitos de engenharia de software ágil e interfaces responsivas.

Palavras-chave: Educação Tecnológica; Gamificação; Repetição Espaçada; Engenharia de Software.

Abstract

Nex_TI is an educational web system focused on Active Study methodologies, Spaced Repetition (SM-2 Algorithm), and Gamification. The goal is to make technological learning more engaging by rewarding the student with Experience Points (XP) and Coins as they progress. The project integrates agile software engineering concepts and responsive interfaces.

Keywords: Educational Technology; Gamification; Spaced Repetition; Software Engineering.

1. Introdução

O avanço das tecnologias digitais ampliou a presença de plataformas educacionais que apoiam o processo de ensino e aprendizagem através de recursos interativos e análise de dados. Neste cenário, o Nex_TI surge como uma plataforma voltada à avaliação e apoio à aprendizagem, integrando funcionalidades que vão além do simples questionário, aplicando a gamificação e a repetição espaçada para maximizar a retenção de conhecimento.

2. Desenvolvimento do Projeto

Abaixo estão detalhadas as 9 etapas de desenvolvimento técnico exigidas para a implementação da plataforma.

Etapa 1: Definição do Negócio Fictício

Denominação: Nex_TI Educacional.

Segmento: Plataforma Web de Avaliação e Apoio à Aprendizagem (EdTech).

Público-alvo: Estudantes de tecnologia, universitários e autodidatas.

Problema Solucionado: A falta de envolvimento no estudo autónomo e a dificuldade de acompanhar o desempenho académico em tempo real através de feedback inteligente.

Etapa 2: Engenharia de Software Ágil Aplicada

O desenvolvimento do software foi guiado pelo framework Scrum. O Backlog do Produto foi estruturado em Épicos e Histórias de Utilizador (User Stories), com estimativas em Story Points baseadas na sequência de Fibonacci.

Exemplo: MD_ÉPICO 3 - Flashcards e Estudo Ativo

Prioridade: Alta 13 Points

US04: Como aluno, quero estudar utilizando o método de repetição espaçada (Anki), para focar nas cartas em que tenho mais dificuldade e otimizar o meu tempo.

Etapa 3: Modelagem de Banco de Dados e NoSQL

Para o Produto Mínimo Viável (MVP) entregue neste semestre, a persistência e o tráfego de dados foram implementados no lado do cliente utilizando a Web Storage API (LocalStorage) e ficheiros estruturados em JSON. Isto garantiu a validação rápida da interface e da lógica de gamificação sem custos de servidor.

Contudo, a modelagem conceptual e lógica para um Banco de Dados Relacional definitivo (ex: SQL Server) já foi projetada teoricamente e mapeada no Roadmap do sistema para implementação futura, visando a escalabilidade.

Etapa 4: POO em C#

Atualmente, a prova de conceito do sistema — incluindo a lógica de negócios, o motor do Algoritmo SM-2 e o sistema de economia virtual (XP/Moedas) — opera de forma fluida através de JavaScript (ES6+).

Para cumprir o ciclo completo de Engenharia de Software a longo prazo, a migração desta lógica para um Back-end robusto em C# (API RESTful) está sob análise técnica. Esta transição aplicará os paradigmas de Programação Orientada a Objetos (encapsulamento, herança e polimorfismo) como requisito obrigatório para as futuras fases de implantação do sistema.

Etapa 5: Desenvolvimento Web Responsivo

As interfaces foram construídas utilizando HTML5 semântico, CSS3 (com Flexbox e Grid Layout) e JavaScript Vanilla. A abordagem adotada foi o Mobile First, garantindo que o sistema se adapte perfeitamente a ecrãs de smartphones, tablets e monitores desktop ultrawide, sem quebra de layout.

Etapa 6: UX e UI Design

A identidade visual adotou um estilo Dark Mode cibernético para reduzir a fadiga visual durante os estudos noturnos. A jornada de navegação foi validada através de Diagramas de Casos de Uso, garantindo que os fluxos para Alunos, Tutores e Administradores sejam intuitivos e diretos.

Etapa 7: Machine Learning e Análise de Dados

A plataforma incorpora o conceito de um "Agente Especialista". Embora não utilize IA Generativa complexa nesta fase, o sistema recolhe e analisa o histórico de acertos e erros do aluno (nos simulados ENADE e flashcards) aplicando regras heurísticas para gerar um feedback motivacional automático e preciso sobre as suas dificuldades reais.

Etapa 8: Comunicação, Liderança, Negociação e LIBRAS

A comunicação da equipa foi mantida através de cerimónias ágeis (Daily Scrum e Sprint Planning). Para assegurar a inclusão, o sistema conta com um widget de acessibilidade (ajuste dinâmico de fontes de 80% a 150% e Modo de Alto Contraste para baixa visão), além da previsão de inclusão futura de suporte a LIBRAS nos vídeos tutoriais da plataforma.

Etapa 9: Integração e Avaliação Final

A arquitetura atual integra com sucesso a interface gráfica à regra de negócios simulada no navegador. O uso do Diagrama de Classes e do Diagrama de Sequência provou que o fluxo de gamificação (responder carta -> calcular intervalo -> atribuir XP) é sólido. A principal limitação atual é a ausência do banco de dados relacional (SQL), que será suprimida nas próximas iterações do software.

3. Conclusão

O desenvolvimento do Nex_TI consolidou a aplicação prática dos conhecimentos de Engenharia de Software Ágil e desenvolvimento Front-end. A estratégia de entregar um MVP web responsivo provou ser eficaz para validar a ideia de gamificação e repetição espaçada na educação. A estrutura lógica desenhada garante que a futura migração para C# e Banco de Dados Relacional ocorra de forma segura e planeada, atestando a maturidade técnica da equipa.
